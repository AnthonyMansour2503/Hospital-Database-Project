# Hospital Database System Project

## Overview
This project involves the design and development of a relational hospital database system intended to simulate a real-world healthcare environment. The project focuses on database design, data modeling, SQL querying, and business intelligence reporting to support operational and analytical decision-making.

The system stores and manages information related to patients, doctors, appointments, prescriptions, billing, medical records, and hospital departments.


## Objectives
- Design a structured relational database for hospital operations  
- Apply data modeling and normalization concepts  
- Generate realistic sample data for testing and analysis  
- Use SQL queries to answer business and operational questions  
- Create a business intelligence dashboard to visualize insights  


## Tools & Technologies
- MySQL  
- SQL  
- Python  
- Faker (Python Library)  
- Power BI  
- Data Modeling / ER Diagrams  


## Database Design

The database includes 11 relational tables:

- Patients  
- Doctors  
- Departments  
- Appointments  
- Prescriptions  
- Medicines  
- Medical Records  
- Billing  
- Staff  
- Rooms  
- Room Assignments  

Relationships between tables were designed using relational database principles and entity relationship modeling.

## Data Generation
Python and the Faker library were used to generate realistic sample data for each table, allowing the database to simulate real-world hospital operations and support analytical querying.

---

## SQL Analysis

SQL queries were created to answer business and operational questions such as:

- Average appointments per patient  
- Revenue generated by department  
- Most prescribed medications  
- Average patient age  
- Most common diagnoses  

The project demonstrates the use of filtering, joins, aggregation, grouping, and analytical querying techniques.

## Business Intelligence Dashboard

A Power BI dashboard was developed to visualize operational insights and support data-driven decision-making.

### Dashboard Features
- Department revenue analysis  
- Patient and appointment trends  
- Prescription statistics  
- Hospital operational metrics  
- Interactive filters and visualizations
